# Office 对象语义与意图选择

> 状态：阶段一已完成；阶段二至阶段四为后续产品方向
>
> 本文定义 Agentic Office UI 下一阶段的人机交互目标，不是数据协议。当前 `0.6.0` 源码候选已提供跨格式引用合同、短生命周期选择状态、基础选择原语和四种 Surface 的阶段一统一选择事件；阶段二可见对象、图层层级和不可见行为选择仍待实现。

## 1. 产品命题

Agent 通常能够通过代码读取和修改 Office 文件，但人类很难只靠语言准确说明“我指的是哪里”。Agentic Office UI 要解决的不是模型推理或 Agent 编排，而是人与 Agent 之间的指向摩擦：

> 系统负责把 Office 内容变成可选择对象，用户通过选择对象表达意图。

语言说明“要做什么”，选择说明“对什么做”。整个产品能力可以归纳为两个系统：

1. **对象系统**：识别画面中有什么、对象之间是什么关系，以及识别结果有多可靠。
2. **选择系统**：让用户从普通文字选择逐步进入对象、图层和行为选择。

具体字段、坐标和序列化协议不在本文定义。

### 1.1 当前能力边界

“已经渲染”不等于“已经成为可引用对象”。`0.6.0` 源码候选的实际基础如下：

| 格式 | 当前已有 | 尚未形成的产品能力 |
|---|---|---|
| DOCX | 搜索、精确文字、页面与区域统一引用，以及编辑器内部的段落、单元格和图片交互 | 阶段二段落、标题、列表、表格、图片与批注对象 |
| XLSX | 搜索、单元格/范围、行列、工作表与区域统一引用，以及控制器内部的图表和图片选择 | 阶段二表格、图表、图片、批注、形状和控件对象交互 |
| PPTX | 搜索、幻灯片、对象内精确文字、区域及带稳定身份的可见对象引用 | 表格单元格、图表内部层级、完整对象矩阵与不可见行为 |
| PDF | 搜索、基于字符几何的精确文字、页面与区域统一引用 | 图片、表格、链接、表单和批注对象层 |

区域框选和截图标注均属于目标能力：区域框选说明“文档中的这一块”，截图标注是在该区域之上补充箭头、矩形或文字等视觉说明。`0.6.0` 源码候选已有格式无关的区域选择器、标注数据合同和四种 Surface 的区域确认事件；截图位图采集和标注编辑器仍由宿主或后续可插拔能力提供。

## 2. 对象系统

### 2.1 什么应该成为对象

不是每个 OOXML 节点、PDF 字形或矢量路径都需要成为用户对象。只有同时满足以下条件的内容，才值得独立选择：

- 人能够在界面中看见或理解它；
- 人可能单独询问、修改、删除、替换或引用它；
- Agent 能够把它作为一个相对独立的操作目标。

面向用户的对象分为三类：

| 类别 | 含义 | 示例 |
|---|---|---|
| 可见内容对象 | 直接出现在文档画面中 | 文字、单元格、图片、表格、图表、形状 |
| 结构容器对象 | 组织或包含其他对象 | 段落、章节、工作表、幻灯片、组合、母版 |
| 行为与设置对象 | 影响内容，但不一定直接可见 | 公式、条件格式、数据验证、动画、翻页效果、链接动作、隐藏状态 |

底层实现细节不应直接暴露给用户。例如 DOCX 的 run、PDF 的 glyph 和矢量 path 都应归入文字、图形或页面区域，而不是要求用户理解文件内部术语。

### 2.2 对象识别来源与可信度

“准确度”不能只用一个分数表达。对象类型、边界、层级和修改后的可重定位性可能分别具有不同可信度。例如用户框出的 PDF 表格区域位置非常准确，但程序未必能够确认它是表格。

产品需要区分四种识别来源。它们不是简单的高低排序：

| 识别来源 | 判断依据 | 示例 | 产品含义 |
|---|---|---|---|
| 原生对象 | 文件明确记录对象身份和位置 | XLSX 单元格、PPTX 形状、DOCX 表格 | 身份和初始边界来自原文件，跨版本仍需校验 |
| 结构推断 | 根据文档结构、样式或关系判断 | DOCX 标题、列表项、段落 | 可信，但需要内容证据校验 |
| 视觉识别 | 根据版面、OCR 或图像判断 | PDF 表格、扫描文字块、图表区域 | 属于推断，不宣称绝对准确 |
| 人工指定 | 用户直接框选或标注 | “就是页面右下角这里” | 忠实记录用户画出的区域，不自动证明语义边界 |

程序需要分别判断：

- 对象类型是否可靠；
- 对象边界是否可靠；
- 父子关系和层级是否可靠；
- 文件修改后是否仍能定位到同一对象。

不同的不确定性需要不同的产品反馈：

- 类型不可靠时保留区域，但不强行命名对象；
- 边界不可靠时显示可调整的候选边界；
- 层级不可靠时列出同一位置的候选对象；
- 文件已经变化、原对象无法确认时，将引用标记为失效并请用户重新选择。

系统不应因为识别不完整而阻止用户表达，也不应假装精确。目标产品始终以页面区域作为无法识别对象时的兜底。

### 2.3 对象是有层级的

同一个点击位置往往同时属于多个对象。选择系统必须允许用户在子对象和父对象之间移动，而不是强迫系统只给出一个答案。

典型层级包括：

- DOCX：文字 → 句子 → 段落或列表项 → 章节 → 页面；
- XLSX：单元格 → 范围 → 表格 → 工作表；
- PPTX：文字 → 文本框或形状 → 组合 → 幻灯片；
- PDF：文字 → 文本行或版面块 → 页面区域 → 页面。

层级选择既解决“点得太细”，也解决“点到了外层对象但用户实际想说内部内容”的问题。

## 3. 选择系统

以下内容描述目标交互。选择能力按照学习成本和表达能力逐步增强，高级能力不替代普通选择，而是在用户需要时向上展开。

这里的四个“级别”描述可以同时存在的交互能力；第 5 节的四个“阶段”描述建议的实现顺序，两者不是同一概念。

### 3.1 第一级：普通内容选择

默认交互延续用户已有习惯：

- DOCX 和 PDF 拖选文字；
- XLSX 点击单元格或拖选范围；
- PPTX 拖选文本框中的文字；
- 对无法识别的内容，目标能力允许直接框选页面区域。

这一层不要求用户进入新的工作模式，适合最常见的解释、改写、翻译和数据询问。

### 3.2 第二级：对象模式

对象模式接近 Figma 或 Photoshop 的选择体验：

- 鼠标经过时显示可选对象轮廓和类型；
- 点击图片、图表、表格、形状、文本框等对象；
- 用简短标签说明当前选中的单个对象，例如“幻灯片 5 · 图表”。

用户不需要知道对象在文件格式中的正式名称，只需要看到它并点到它。对象技术名称和识别依据由系统负责解释给 Agent。

### 3.3 第三级：层级与重叠选择

这一层解决对象嵌套、遮挡和背景对象难以命中的问题。基础交互可以包括：

- 连续点击轮换同一位置的候选对象；
- `Alt/Option + 点击` 穿透当前对象；
- 长按或右键显示“此位置包含的对象”；
- 对象面包屑；
- 图层面板；
- 父级、子级和同级对象切换。
- 进入表格、图表或组合内部选择子对象；
- 同时选择多个对象，并标记目标、参考或插入位置等角色。

完整层级路径可以表示为：

```text
第 3 页 › 表格 2 › 第 4 行 › 第 2 个单元格
```

```text
幻灯片 5 › 组合 1 › 图表 › 系列“收入”
```

在此基础上，可以进一步提供 **图层透镜**：将页面中的对象按层级和 Z 轴顺序展开，让用户看见被遮挡的对象、组合结构和前后关系。

以上是宿主可以构建的产品交互，不要求组件库提供固定的菜单、面包屑、图层面板或角色托盘。组件库只需要让这些界面消费同一对象树、候选、引用和选择事件。

3D 爆炸视图是图层透镜的一种高级表现形式。它具有很强的展示和传播能力，但不是对象系统成立的前提。产品应先完成稳定的对象层级、候选命中和图层面板，再决定是否增加可旋转、倾斜的 3D 视图。

### 3.4 第四级：行为与设置选择

Office 文件中还有一类无法通过普通画布点击表达的目标：公式、动画、翻页效果、条件格式、隐藏状态等不可见行为。

这些内容应该成为一等可选择对象，但不应强行伪装成画布对象。产品应根据其语义，通过时间线、属性面板、结构树或界面标记完成选择。

以 PPTX 为例：

- 翻页效果属于两张幻灯片之间，可在缩略图之间或切换面板中选择；
- 对象动画属于“对象 + 时间线效果”，可通过对象旁的动画标记或时间线条目选择；
- 动画顺序在时间线中选择；
- 母版和版式对象在结构树或图层面板中选择；
- 超链接、点击动作和媒体控制通过对象上的行为标记选择；
- 演讲者备注在备注区域中选择。

其他格式中的不可见目标包括：

| 格式 | 行为与设置对象示例 |
|---|---|
| DOCX | 段落样式、动态字段、书签、分节、修订状态、批注状态、交叉引用 |
| XLSX | 公式、条件格式、数据验证、命名区域、隐藏行列、保护规则 |
| PPTX | 对象动画、动画顺序、翻页效果、母版、版式、动作和备注 |
| PDF | 链接动作、表单规则、批注状态、附件、页面图层、数字签名和权限信息 |

上表描述可继续扩展的候选对象，不等于首批交付承诺。首批范围只以第 4 节矩阵为准。

不可见对象选择使用户不仅能问“这里写了什么”，还可以问“为什么会这样显示”“点击以后为什么跳到这里”“这页为什么以这种方式出现”。

## 4. 首批能力覆盖与阶段归属

为了尽快形成完整体验，首批产品范围不追求识别所有 Office 特性，而是随四个实现阶段逐步累积。阶段一完成基础指向，阶段二把可以直接命中的高频可见内容作为单个整体对象，阶段三处理对象内部层级和遮挡，阶段四再覆盖不可见行为与设置。

矩阵中的对象是对应阶段的首批交付范围。阶段完成意味着四种格式中列出的对象均已实现并通过该阶段验收；如果后续决定缩减范围，必须先更新本文，而不能以实现了少数示例对象替代阶段完成。

同一个 Office 特性可以同时具有可见承载物和不可见行为：阶段二选择用户看得见的内容，阶段四选择它背后的动作、规则或状态。例如阶段二选中链接文字，阶段四选择链接的跳转目标；阶段二选中表单控件，阶段四选择它的校验规则。

| 格式 | 阶段一：基础选择 | 阶段二：单个可见对象 | 阶段三：层级与多对象 | 阶段四：行为与设置 |
|---|---|---|---|---|
| 通用 | 页面、幻灯片、工作表、文字范围、任意页面区域 | 可见内容块和嵌入对象 | 重叠候选、父子层级和多目标角色 | 不可见设置及其作用范围 |
| DOCX | 文字范围、页面区域 | 段落、标题、列表项、表格、图片，以及批注标记和内容 | 表格行与单元格、嵌套表格、结构层级和多目标关系 | 样式、动态字段、书签、分节、修订状态、批注状态和交叉引用 |
| XLSX | 单元格、范围、行列 | 表格、图表、图片，以及批注标记和内容 | 图表系列与数据点、对象重叠和多目标关系 | 公式、条件格式、数据验证、命名区域和保护规则 |
| PPTX | 幻灯片、文字范围、页面区域 | 文本框、形状、图片、表格和图表 | 单元格、图表内部元素、组合、遮挡和多目标关系 | 动画、翻页效果、动作、母版和备注 |
| PDF | 文字范围、页面区域 | 文字块、图片、表格、链接锚点、表单控件，以及批注标记和内容 | 表格行与单元格、版面块层级和重叠标记 | 链接动作、表单规则、批注状态、附件、页面图层、数字签名和权限信息 |

在目标体验中，某个对象暂时无法识别时，用户仍可通过区域框选和截图标注表达意图。这样产品能力可以逐步扩展，而不会因为长尾格式特性阻塞基本闭环。

## 5. 产品演进

### 阶段一：MVP——选中内容

- 统一页面、幻灯片、工作表、文字、单元格范围和页面区域选择；
- 显示当前选择范围，并说明它来自原生结构、推断还是人工区域；
- 完成“单个目标 + 用户指令 + Agent 修改 + 结果确认”的最小闭环；
- 文件更新后尝试恢复原目标，无法确认时明确提示引用失效；
- 对识别失败提供明确降级，不制造虚假精确。

### 阶段二：点中对象

- 提供对象模式、悬停轮廓和单对象标签；
- 优先支持四种格式的高频可见对象；
- 本阶段只要求把直接命中的可见内容作为一个完整对象，不要求进入表格、图表或组合内部，也不要求在父子层级间导航。

### 阶段三：看见层级

- 提供重叠候选、穿透选择、面包屑和图层面板；
- 支持进入对象内部、组合、父子关系、多目标选择和角色标记；
- 在对象层级稳定后探索 3D 图层透镜。

### 阶段四：选择行为

- 让第 4 节矩阵列出的公式、条件格式、动画、切换和其他首批不可见设置成为可选择目标；
- 通过时间线、属性面板和结构树表达其作用范围；
- 支持用户向 Agent 询问原因、修改行为或复用设置。

## 6. 典型任务闭环

MVP 以单目标任务形成最小闭环。例如用户希望“把这段话改得更简洁”：

1. 用户在 DOCX Surface 中拖选目标文字；
2. 界面清楚显示文字范围和识别来源；
3. 用户输入简短修改指令；
4. 宿主把简短指令和对象引用交给其 Agent；
5. Agent 使用宿主选择的文档工具生成新文件；
6. Surface 重新加载结果，并尽可能回到原对象位置供用户确认。

阶段三完成后，同一闭环可以扩展为多目标任务，例如选择一张图片作为修改目标，再选择旁边卡片作为样式参考。

MVP 闭环是否成立，只使用阶段一标准判断：

- 用户不需要拼装页码、坐标和长篇上下文；
- 用户可以选择页面、幻灯片、工作表、文字、单元格范围、行列和页面区域；
- 用户始终知道当前选中了哪个基础目标；
- 系统能够说明选择来自原生结构、推断还是人工区域；
- 识别失败不会丢失用户意图，也不会伪装成精确对象；
- 文件更新后，系统能够恢复目标或明确要求重新选择；
- 宿主可以完成“选择—指令—Agent 修改—结果确认”的闭环。

后续阶段分别增加自己的验收标准：

- 阶段二：对象模式不能让默认文字和单元格选择变难，用户能够看清单个对象的轮廓与类型；
- 阶段三：用户能够理解当前对象的层级、处理遮挡，并明确多个对象各自承担的角色；
- 阶段四：用户能够看见行为或设置的作用范围，并将其作为独立目标交给 Agent。

每个阶段还必须覆盖第 4 节矩阵中该阶段列出的全部首批对象，以上体验标准不能替代格式与对象覆盖验收。

指针不是唯一入口。对象选择后续还应支持键盘切换候选、触控长按和可访问的对象列表，避免高级选择只能依赖精细鼠标操作。

## 7. 产品原则

1. **默认交互必须熟悉。** 高级对象能力不能破坏文字和单元格的普通选择体验。
2. **可见性优先于术语。** 用户先看到并点到对象，系统再说明它是什么。
3. **不假装精确。** 原生结构、推断结果和人工区域必须明确区分。
4. **对象具有层级。** 同一位置允许选择文字、容器、组合或页面，而不是只返回单一命中。
5. **区域选择是目标兜底。** 识别失败时仍应能够表达“就是这里”。
6. **不可见行为也是对象。** 公式、动画和设置应通过适合其语义的界面选择。
7. **视觉创新服务于选择。** 3D 图层视图的价值是降低遮挡和层级理解成本，而不是单纯展示效果。
8. **组件库不替宿主编排意图或调用 Agent。** 本库提供对象识别、稳定引用和选择事件；引用集合、业务角色、模型调用与文档修改工具仍由宿主负责。
