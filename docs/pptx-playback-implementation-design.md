# PPTX 播放正式实现设计

## 1. 文档状态

- 状态：已实现并通过自动验收；公开发布前待独立复核
- 日期：2026-07-11
- 适用范围：`@arcships/pptx-core`、`@arcships/pptx-core/browser`、`@arcships/vue-pptx`
- 静态底座：固定 `@aiden0z/pptx-renderer@1.2.4`
- 前置证据：[PPTX 播放探索版设计](pptx-playback-exploration-design.md)、[正式实现前的三个代码验证](../apps/pptx-playback-lab/results/preimplementation-validation.md)
- 公开接口附录：[PPTX 播放类型与接口](pptx-playback-api-design.md)
- 兼容与验收：[PPTX 播放兼容范围与验收](pptx-playback-compatibility-and-acceptance.md)
- 实施顺序：[PPTX 播放工程化路线图](pptx-playback-roadmap.md)

探索阶段已经结束。本设计取代 [PPTX 预览与播放技术设计](pptx-preview-playback-design.md) 中尚未验证的播放模型，作为正式播放实现的唯一技术入口。静态预览现有接口继续有效；播放能力不进入当前 `0.2.0` 候选版本。

## 2. 结论

正式播放使用三个已经通过代码验证的基础：

1. 稳定对象身份：同页动画使用精确对象编号，Morph 使用分级跨页身份；
2. 时间节点树：保留 `cTn`、父子关系、条件和事件，不再把扁平批次作为主模型；
3. 属性轨道：先合并透明度、位移、缩放、旋转、裁剪、颜色和媒体控制，再创建浏览器动画。

静态渲染仍由 `@aiden0z/pptx-renderer` 完成。项目内新增一次解包会话、播放解析、能力报告、播放控制器和 Vue 演示界面。探索代码保留为证据，不直接迁入正式包。

## 3. 目标与非目标

### 3.1 第一版目标

- 在浏览器客户端打开、预览和播放 PPTX；
- 同一个文件只解包一次，静态模型和播放模型共享结果；
- 支持基础时间顺序、批准的对象动画、页面切换、文字逐段、对象触发、自动换页和音视频控制；
- 暂停、继续、重置、上一步、下一步和跳页得到确定结果；
- 支持范围内的动画只作用于精确目标；
- 未支持内容保持静态可见，并产生结构化能力报告；
- 继续保持纯类型入口不访问浏览器对象；
- 不破坏现有静态预览 API。

### 3.2 第一版非目标

- 编辑或保存 PPTX；
- 宏、ActiveX、OLE 和嵌入程序执行；
- 图表和 SmartArt 内部逐项动画；
- 逐字和逐词动画；
- 三维页面切换的原效果还原；
- 没有强身份时宣称 Morph 对象已准确匹配；
- 服务端转图片、PDF 或调用 Office；
- 多人同步播放和演讲者协作。

## 4. 设计依据

探索结果已经证明：

- 三种常用开始方式、淡入淡出、缩放、擦除、文字逐段、对象点击、媒体、书签、自动换页和暂停继续都有可执行路径；
- `!!` 显式名称和相同创建编号可以稳定建立跨页身份；
- 最小时间节点树在文字、书签和自动往返素材中达到 6/6、3/3、1/1 效果对应，顺序一致；
- 同对象淡入淡出直接并行会产生 `opacity=1`、`visibility=hidden` 的矛盾状态，属性轨道可以消除该问题；
- 43.4 MB、79 页真实业务文件在当前 50 MiB 输入限制内可快速打开；119 MB 文件应被当前限制拒绝，不作为第一版 Worker 前置理由；
- 图表和 SmartArt 内部目标缺少语义编号，不能靠页面元素顺序还原。

PowerPoint 支持通过选择窗格把前后页对象命名为相同的 `!!名称`，强制 Morph 对应。因此严格 Morph 把该名称作为最高优先级：[Morph transition: Tips and tricks](https://support.microsoft.com/en-us/powerpoint/morph-transition-tips-and-tricks)。

## 6. 包边界

### 6.1 `@arcships/pptx-core`

根入口只包含可序列化类型、错误码、能力报告、对象身份、时间节点和属性轨道定义。禁止导入第三方渲染器，禁止访问 `window`、`document`、`DOMParser`、`Worker` 和媒体元素。

建议目录：

```text
packages/pptx-core/src/
├── types.ts
├── playback/
│   ├── types.ts
│   ├── identity.ts
│   ├── time-tree.ts
│   ├── track-compiler.ts
│   ├── capability.ts
│   └── state.ts
├── browser-types.ts
├── browser.ts
└── browser/
    ├── archive-adapter.ts
    ├── renderer-adapter.ts
    ├── playback-parser.ts
    ├── media-resolver.ts
    ├── clock.ts
    ├── controller.ts
    └── session.ts
```

### 6.2 `@arcships/pptx-core/browser`

浏览器入口负责：

- 调用 `office-runtime` 读取来源；
- 使用固定限制调用 `parseZipLazyMedia()`；
- 从同一份 `PptxFiles` 构建静态模型和播放模型；
- 提供渲染器适配、媒体对象地址、播放控制器和文档会话；
- 统一处理取消、旧任务失效和销毁。

第三方类型不出现在公开接口中。`PptxFiles`、`PresentationData` 和渲染句柄只存在于包内部。

### 6.3 `@arcships/vue-pptx`

Vue 包负责：

- 浏览和演示两种界面；
- 舞台尺寸、双页叠放和全屏；
- 键盘、鼠标和触摸操作；
- 显示加载、播放、媒体授权、未支持和错误状态；
- 把公开控制器暴露给组件调用方。

Vue 包不解析 XML，不计算时间节点，不合并动画属性，也不保存第二份播放状态。

## 7. 单次解包与文档会话

当前静态预览通过第三方查看器的便捷 `open()` 隐藏了解包结果。正式实现改为显式会话：

1. `office-runtime` 读取来源并执行 50 MiB 输入限制；
2. `PptxArchiveAdapter` 调用一次 `parseZipLazyMedia()`；
3. 从同一个 `PptxFiles` 调用 `buildPresentation()`；
4. 播放解析器读取同一份页面 XML、关系和媒体；
5. 渲染器适配器加载已经构建的 `PresentationData`；
6. 会话销毁时统一释放页面、媒体、对象地址和播放控制器。

`createPptxPreviewSession()` 保留，并改为对新文档会话的兼容包装。新功能使用 `createPptxDocumentSession()`。

第一版不把完整解包和第三方模型搬入 Worker。理由是当前 50 MiB 限制内真实 43.4 MB 文件已验证可接受，而第三方模型跨线程传输会引入第二套模型。若正式三轮基线超过性能门槛，只把可序列化的页面 XML 扫描和播放模型编译迁入 Worker，不重复解包。

## 8. 对象身份与渲染器合同

### 8.1 同页精确身份

同一页动画目标只使用精确身份：

```text
页面部件路径 + 来源层级 + 组合路径 + cNvPr/@id
```

例如：

```text
ppt/slides/slide4.xml|slide|group:8/shape:12
```

`spTgt/@spid` 只在当前页面和来源范围内查找。找不到或找到多个目标时不执行该效果，并记录警告。禁止使用 DOM 顺序、数组下标、对象文字或相邻元素猜测同页目标。

### 8.2 DOM 标记

固定渲染器补丁必须递归写入：

```html
data-pptx-slide-path="ppt/slides/slide4.xml"
data-pptx-node-id="12"
data-pptx-group-path="8"
data-pptx-object-key="ppt/slides/slide4.xml|slide|group:8/shape:12"
```

顶层形状、图片、表格、图表、组合和组合子对象都使用同一规则。版式和母版对象增加 `layout` 或 `master` 来源。补丁以 `pnpm patch` 固定在仓库中，并同时向上游提交；正式实现不等待上游发布。

文字段落另外写入 `data-pptx-paragraph-index`。它表示文字对象内部从零开始的段落编号，用来对应时间节点中的 `pRg`；段落标记必须附着在实际参与显示和隐藏的元素上，不能在播放时临时按页面元素顺序推断。

### 8.3 Morph 跨页身份

Morph 匹配只发生在前后页对象之间，优先级如下：

1. 相同 `!!` 显式名称：强匹配；
2. 相同创建编号：强匹配；
3. 相同普通名称、对象类型和接近几何：中等匹配；
4. 相同文字、对象类型和接近几何：弱匹配。

严格模式只执行强匹配。近似模式可以执行中等匹配；弱匹配只用于调试报告，不自动把对象当成同一对象。每个匹配结果必须保存依据、分数和是否唯一。

## 9. 播放文档与时间节点树

播放解析结果按文档、页面和节点组织：

```text
PptxPlaybackDocument
└── PptxPlaybackSlide
    ├── objects
    ├── morphFromPrevious
    ├── transition
    ├── rootTimeNode
    │   ├── sequence / parallel
    │   ├── effect
    │   ├── media
    │   ├── command
    │   └── unknown
    ├── media
    ├── actions
    └── capabilityReport
```

每个时间节点保留：

- XML 节点编号和容器类型；
- `nodeType`、持续时间、延迟、重复、自动往返、填充和重启方式；
- 开始、结束、上一项和下一项条件；
- 条件事件、目标对象、书签名称和引用节点；
- 子节点顺序；
- 已识别效果或无法识别的原始摘要。

解析器不能只返回“单击批次”。批次是控制器从节点树计算出的临时视图，用于界面显示和下一步操作。

## 10. 时间调度

### 10.1 中央时钟

播放控制器维护一个中央时钟，记录：

- 当前页面播放编号；
- 当前时间位置；
- 播放、暂停和等待单击状态；
- 已激活、运行中和已完成节点；
- 等待的条件和媒体书签；
- 活动浏览器动画与媒体。

不允许每个效果各自永久持有不可追踪的 `setTimeout`。中央时钟使用 `performance.now()` 和 `requestAnimationFrame` 推进条件；浏览器动画负责已经开始的插值。暂停时同时暂停时钟、浏览器动画和受控媒体。

### 10.2 条件执行

- `on-click`：等待普通下一步操作；
- `with-previous`：和同一触发组的前一效果使用相同开始时间；
- `after-previous`：前一效果完成后加延迟；
- `on-shape-click`：指定对象点击后激活；
- `on-media-bookmark`：媒体时间越过书签后激活，每次页面播放只触发一次；
- `on-begin`、`on-end` 和节点引用：由被引用节点状态触发；
- 无法理解的条件：节点不执行，能力报告记录条件和原始节点编号。

### 10.3 自动换页

页面切换完成后启动本页自动节点。PowerPoint 的“之后自动换页”计时从本页最后一个动画或其他效果结束后开始，因此控制器在自动时间树稳定结束后再启动 `advanceAfterMs` 计时：[Set the timing and speed of a transition](https://support.microsoft.com/en-us/powerpoint/set-the-timing-and-speed-of-a-transition)。

若页面存在无限重复且没有可到达的结束状态，自动换页不启动，并产生 `AUTO_ADVANCE_BLOCKED_BY_INDEFINITE_NODE`。同时允许单击和自动换页时，先发生者生效。

## 11. 属性轨道

### 11.1 轨道类型

第一版定义以下独立属性：

- `display`：可见、隐藏和命中；
- `opacity`；
- `translateX`、`translateY`；
- `scaleX`、`scaleY`；
- `rotate`；
- `clipPath`；
- `filter`；
- `fillColor`、`lineColor`、`textColor`；
- `mediaPlayback`、`mediaTime`、`mediaVolume`。

静态定位的 `transform` 不作为动画轨道。位移、缩放和旋转使用 CSS 独立属性或渲染器提供的动画外层，避免覆盖静态布局。

### 11.2 编译过程

1. 从渲染对象读取并冻结静态初始值；
2. 把时间节点展开成绝对开始时间和结束时间；
3. 把每个效果转换为一个或多个属性片段；
4. 收集所有属性片段的开始、结束和关键点；
5. 对同一对象、同一属性进行冲突裁决；
6. 输出可求值的属性轨道；
7. 由浏览器适配器创建动画。

### 11.3 冲突规则

- 不同属性直接组合；
- 同一属性后激活的节点在其活动区间内优先；
- 同时激活时按时间节点中的稳定顺序决定，后出现者优先，并记录 `SAME_PROPERTY_TIE_BREAK`；
- 优先节点结束后，根据 `fill`、保持和恢复规则决定保留结果，或重新露出仍活动的底层片段；
- `display` 和 `opacity` 一起求值，禁止一个效果单独留下隐藏状态、另一个效果留下不透明状态；
- 重置、上一步和跳转不反向猜测动画，而是重新求值目标时间点的全部轨道。

探索中的“同时淡入和淡出”已经证明该方法可以得到唯一最终状态。正式版必须继续覆盖不同时刻重叠、重复和自动往返。

## 12. 动画执行与页面状态

每页渲染完成后执行：

1. 保存静态样式快照；
2. 编译本页属性轨道；
3. 在不播放的情况下求值时间零；
4. 应用进入动画的初始状态；
5. 页面切换完成后启动自动节点；
6. 等待用户或条件触发后继续。

未支持进入动画不能把对象隐藏。只有已经成功编译出轨道的进入效果才修改时间零状态。

上一步采用重建方式：取消活动动画，恢复静态样式，从时间零快速求值到前一个单击边界，再恢复等待状态。该操作不尝试倒放媒体或逐个反转浏览器动画。

## 13. 页面切换与 Morph

页面切换期间旧页和新页同时存在：

1. 渲染新页但不启动页内动画；
2. 应用新页时间零状态；
3. 执行双页切换；
4. 释放旧页渲染句柄和媒体；
5. 启动新页自动节点。

第一版严格支持直接切换、淡化、推进和擦除。其他二维切换在近似模式下可以使用明确标注的退路。三维切换默认保持静态直接切换并报告未支持。

Morph 的强匹配对象执行几何、透明度和已批准样式的补间；旧页未匹配对象淡出，新页未匹配对象淡入。弱匹配不进入严格模式。组合内部 Morph 等待渲染器暴露组合子对象身份。

## 14. 媒体与书签

媒体解析器从页面关系和 `p14:media` 读取：

- 视频或音频关系；
- 自动播放、播放、暂停、停止和切换暂停命令；
- 裁剪开始和结束；
- 循环和音量；
- 媒体书签；
- 书签触发的时间节点。

媒体通过会话创建受控对象地址。离开页面、换文件和销毁会话时必须暂停媒体并撤销地址。

渲染器只输出媒体占位时，`PptxRendererAdapter` 根据精确对象键在占位位置挂载受控的 `audio` 或 `video` 元素，并继承原对象的尺寸、裁剪、层级和命中区域。媒体元素仍属于当前页面渲染句柄，不进入静态模型，也不能绕过会话单独存活。

浏览器拒绝无用户操作播放时，控制器进入 `blocked` 媒体状态并发出事件；Vue 显示“点击开始播放”，用户点击后恢复等待中的媒体和时间节点。外部媒体默认禁用，只报告其存在。

媒体书签按媒体时间触发。向前跳过书签时触发一次；向后拖动到书签前会重新允许触发。页面播放编号变化后清空全部书签状态。

## 15. 播放控制器

播放状态为：

```text
idle → ready → transitioning → running → waiting → ended
                    │             │          │
                    └── paused ───┴── blocked
任意活动状态 → disposed
```

主要操作：

- `next()`：执行下一普通单击组；本页没有普通单击组时进入下一张非隐藏页；
- `activateObject(objectKey)`：执行该对象的专用触发或内部动作；没有专用行为时返回 `false`，由界面决定是否继续调用 `next()`；
- `previous()`：当前页存在已完成单击组时重建到前一边界，否则进入上一张非隐藏页；
- `play()`：启动当前页可自动执行的节点；
- `pause()`、`resume()`：同时控制时钟、动画和媒体；
- `reset()`：恢复当前页时间零；
- `goToSlide()`：取消当前页并直接进入指定页；
- `dispose()`：使所有异步回调失效并释放资源。

切换进行中只允许保存一个待执行导航命令，后来的命令覆盖先前命令，避免快速连击创建多层页面。

## 16. 隐藏页、动作与链接

- 浏览模式显示隐藏页标记并允许直接打开；
- 演示模式的顺序前进和后退跳过隐藏页；
- 指定页跳转和内部动作可以进入隐藏页；
- 页面对象点击先调用 `activateObject()`；只有它返回 `false` 时，演示界面才把同一次点击作为普通下一步；
- 内部跳页由控制器执行；
- 外部链接只允许 `https:`、`http:` 和批准的 `mailto:`，默认发出动作事件，由宿主决定是否打开；
- 宏、程序和任意命令不执行。

## 17. Vue 组件设计

现有 `PptxViewer` 默认保持 `browse` 模式。新增 `mode="present"` 后：

- `browse` 纵向连续展示全部幻灯片并按可视区域挂载；
- `present` 只挂载当前页和切换需要的相邻页；
- 显示播放控制条；
- 空格和右方向键调用 `next()`；
- 左方向键调用 `previous()`；
- Home、End 和 Escape 按公开规则处理；
- 全屏必须由用户操作触发；
- 媒体被阻止时显示恢复入口；
- 能力报告通过插槽或事件交给产品界面。

组件通过模板引用暴露控制器，不要求调用方访问内部 session。具体属性、方法和事件见接口附录。

## 18. 能力报告、警告与错误

每个文件输出：

- 发现的页面切换、动画、目标、触发器和媒体数量；
- 严格执行、近似执行、静态保留和无法解析的数量；
- 每项的页面、对象、原始功能和处理方式；
- Morph 匹配依据和可信度；
- 找不到目标、同属性裁决、媒体阻止和无限节点等警告。

能力报告是事实记录，不直接显示“兼容率”。产品如需百分比，分母必须包括所有发现和无法识别的项目。

错误分为加载错误、解析错误、渲染错误、播放错误、媒体错误和资源错误。单个效果失败默认降级为静态并继续页面；文档损坏、资源超限和会话销毁才终止整个会话。

## 19. 性能与资源

第一版继续使用现有默认限制：

- 输入 50 MiB；
- ZIP 条目 4,000；
- 单项解压 32 MiB；
- 总解压 256 MiB；
- 媒体总量 192 MiB；
- 解压并发 8。

演示模式最多保留当前页、旧切换页和一个预加载页。预加载只解析和渲染，不启动时间节点和媒体。

每次页面离开必须释放：

- 页面渲染句柄；
- 浏览器动画；
- 页面事件监听；
- 媒体元素和对象地址；
- 图表实例；
- 待处理导航和时钟任务。

正式性能门槛见兼容与验收文档。超过门槛后先定位解包、建模、渲染或媒体阶段，再决定是否迁入 Worker，不能先引入第二次解包。

## 20. 实现原则

1. 探索代码只复制算法结论，不复制临时状态结构；
2. 先实现类型、解析和事件记录，再接浏览器动画；
3. 每个支持项必须有 PowerPoint 素材和基准结果；
4. 同页目标没有精确身份就不执行；
5. 未支持效果默认静态保留，不伪装成相似效果；
6. 近似模式必须显式开启并报告；
7. 当前固定第三方版本和补丁必须进入锁文件、构建和外部消费测试；
8. 每个阶段只在对应退出条件通过后进入下一阶段。

## 21. 需要在实现中继续确认的细节

以下内容不影响架构，可以在对应功能实现时用素材确定：

- 同属性完全同时开始时，PowerPoint 的精确优先规则；
- 非显式普通对象名称在 Morph 中的可信门槛；
- 曲线路径坐标和页面旋转后的换算；
- 音频裁剪、循环和页面自动换页的组合；
- Safari 和 Firefox 对独立 `translate`、`scale`、`rotate` 的差异；
- PowerPoint 三维切换是否值得逐项实现。

这些是功能级验收问题，不再改变“对象身份、时间节点树、属性轨道”的总体设计。
