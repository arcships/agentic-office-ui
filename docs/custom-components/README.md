# 自定义组件指南

自定义组件有两种方式：

1. 保留官方高层组件，只把控制器和按钮放到外部；
2. 使用公开舞台或 Runtime 自己构建界面。

推荐先选择第一种。公开但已经标记 `@deprecated` 的 DOCX 页面组件和 XLSX 网格、工具栏、浮层只为兼容保留，不应成为新自定义组件的底座。

- [自定义 DOCX 编辑界面](docx.md)
- [自定义 XLSX 界面](xlsx.md)
- [自定义 PDF 界面](pdf.md)
- [自定义 PPTX 播放器](pptx.md)

## 判断标准

| 需求 | 推荐方式 |
|---|---|
| 换按钮、布局和状态文字 | 高层组件 + 控制器/属性 |
| 隐藏官方工具栏 | 使用 `show*` 属性并增加外部按钮 |
| 自己控制 PPTX 舞台 | `PptxStage` + 两个组合函数 |
| 自己解析 DOCX/XLSX/PDF | 对应 Runtime 或核心包 |
| 直接导入 `src/` 或 `dist/` 深层文件 | 不支持 |
